⚠️ **安全警告**：本文档仅供授权的安全测试使用。

## 驱动加载失败：拒绝访问

### 错误信息
```
驱动加载失败：(5, 'OpenSCManager', '拒绝访问。')
```

### 原因
Windows 11 强制要求所有内核驱动必须有有效的数字签名。BYOVD 使用的漏洞驱动通常没有签名，因此被系统阻止。

### 解决方案

#### 方法 1：禁用驱动签名强制（推荐用于测试）

1. **以管理员身份打开 PowerShell**：
   - 右键点击 PowerShell
   - 选择"以管理员身份运行"

2. **启用测试模式**：
   ```powershell
   bcdedit /set testsigning on
   ```

3. **禁用驱动签名强制**：
   ```powershell
   bcdedit /set nointegritychecks on
   ```

4. **重启电脑**

5. **运行程序**

6. **测试完成后恢复**（重要）：
   ```powershell
   bcdedit /set testsigning off
   bcdedit /set nointegritychecks off
   ```

#### 方法 2：高级启动选项（临时禁用）

1. **打开设置** → **更新和安全** → **恢复**

2. **高级启动** → **立即重新启动**

3. **选择**：疑难解答 → 高级选项 → 启动设置 → 重启

4. **按 F7**：禁用驱动程序强制签名

5. **运行程序**

注意：此方法只在本次启动有效，重启后恢复。

#### 方法 3：在虚拟机中测试（最安全）

1. 安装 VMware 或 VirtualBox
2. 创建 Windows 11 虚拟机
3. 在虚拟机中禁用驱动签名强制
4. 在虚拟机中运行测试

## 其他常见问题

### 1. 找不到驱动文件

**错误**：`无法找到驱动文件 BdApiUtil64.sys`

**解决**：
- 确保 `BdApiUtil64.sys` 存在于以下位置之一：
  - 库文件所在目录
  - 当前工作目录
- 或者将驱动文件 base64 编码后嵌入到库中

### 2. 缺少依赖库

**错误**：`缺少依赖库 win32file`

**解决**：
```bash
pip install pywin32 psutil
```

### 3. 权限不足

**错误**：`需要管理员权限`

**解决**：
- 右键点击程序或终端
- 选择"以管理员身份运行"

---

## 安全注意事项

1. **仅在隔离环境测试**：建议使用虚拟机
2. **测试后恢复系统**：禁用测试模式，恢复驱动签名验证
3. **不要禁用安全软件**：保持杀毒软件运行
4. **了解法律风险**：确保有书面授权

| 系统版本 | 驱动签名要求 | 解决方法 |
|----------|-------------|----------|
| Windows 10 | 强制 | 禁用签名强制 |
| Windows 11 | 强制 | 禁用签名强制 |
| Windows 7 | 可选 | 可直接使用 |
| Windows 8/8.1 | 强制 | 禁用签名强制 |
